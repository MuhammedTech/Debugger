About

Bug Tracker is an application to track errors in software. It was created by Mukammed Alimbet as a student project for the UCLA Extension certificate program in "Application Programming". Bug Tracker uses a ticketing system to record bugs or other issues in the software development process on a per-project basis. It implements user- and role-level security to ensure only authorized users can access tickets and projects. The application was built using Python / Flask framework with an sqlite3 database. Ticket priorities

A ticket can be Critical, High, Medium or Low priority.

  • Ticket statuses: A ticket can be in one of five statuses:
    • New: Ticket automatically assigned this status on creation
    • Waiting for support: Ticket automatically changed to this status when it is assigned to a developer
    • Waiting for customer: Ticket is waiting on a response from a third-party source
    • On hold: Work on the ticket has been suspended
    • Resolved: The issue raised in the ticket has been resolved

    • Ticket types: A ticket can be of type Error report, Service request, Feature request or Other.

    FAQ

    How do I submit a new ticket?
    Make sure you are assigned the Submitter role and have been assigned to the project. If not, contact your Admin. Navigate to your Tickets page and click on the Create New link to fill out the ticket details and submit the ticket.
    When are email notifications sent out?
    Email notifications are sent to developers on the following events: Developer assigned to ticket Ticket reassigned to another developer User other than the developer edits ticket User other than developer adds a comment or attachment
    Can I change my user name and password?
    Yes, any user can edit his or her first and last name, email and/or password. Go to Settings on the top menu to change these.
    How can I get my role changed?
    Contact your Admin to change your role.
